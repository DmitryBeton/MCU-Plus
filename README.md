# MCU+

**MCU+** — iOS-приложение для студентов МГПУ. В приложении собраны три ключевых учебных сценария: просмотр расписания, чтение новостей университета и ведение личных учебных заметок. Проект развивается как практический студенческий продукт с упором на современный стек Apple, персонализацию и интеграцию с собственным backend для расписания.

## Что умеет приложение

- Онбординг с выбором института и учебной группы.
- Экран расписания с недельной навигацией и переключением дней свайпами.
- Загрузка расписания из backend по выбранной группе.
- Отображение статусов занятий: `Сейчас`, `Скоро`, `Завершено`, `Отмена`, `Замена`, `Онлайн`.
- Личные задачи и заметки на выбранную дату: создание, отметка о выполнении, удаление.
- Лента новостей МГПУ с загрузкой с сайта, пагинацией и pull-to-refresh.
- Внутренний экран детальной новости внутри приложения.
- Вкладка настроек с выбором языка, темы и адреса backend-сервера.
- Локализация на русский и английский языки.
- Поддержка светлой, тёмной и системной темы.

## Архитектура

Проект построен вокруг экранов SwiftUI и сервисного слоя, который отвечает за загрузку и синхронизацию данных.

- `ContentView` определяет стартовый поток: онбординг или основной интерфейс.
- `OnboardingView` загружает институты и группы из backend и сохраняет выбор через `@AppStorage`.
- `Home` отображает календарный экран расписания и инициирует синхронизацию на выбранный день.
- `ScheduleSyncService` получает данные из внешнего источника и сохраняет их в `SwiftData`.
- `NetworkScheduleDataSource` запрашивает расписание у backend через `GET /api/v1/schedule/`.
- `CatalogService` загружает список институтов и групп через `GET /api/v1/catalog/`.
- `NewsService` загружает и парсит HTML-страницы новостей с сайта МГПУ.

## Источники данных

Сейчас в приложении используются два основных сетевых источника:

- **Расписание**: собственный backend с endpoint-ами:
  - `GET /api/v1/catalog/`
  - `GET /api/v1/schedule/?groupId=...&from=...&to=...`
- **Новости**: сайт МГПУ, HTML-парсинг страницы новостей и детальной страницы статьи.

Дополнительно в проекте сохранён `MockScheduleDataSource`, который можно использовать для локальной разработки или изоляции UI, но по умолчанию приложение работает через backend.

## Экран расписания

Расписание показывается для выбранной учебной группы и выбранной даты. Данные синхронизируются по сети, затем сохраняются в `SwiftData`, чтобы UI работал через локальное хранилище. На одном экране объединяются:

- пары из backend;
- личные пользовательские задачи на тот же день.

Для занятий предусмотрены визуальные состояния:

- `Сейчас` — пара идёт в текущий момент;
- `Скоро` — следующая пара начнётся позже сегодня;
- `Завершено` — пара уже прошла;
- `Отмена`, `Замена`, `Онлайн` — академические статусы, пришедшие с сервера.

## Экран новостей

Новости загружаются напрямую с [mgpu.ru](https://www.mgpu.ru/news/) и отображаются нативным списком. В приложении есть:

- основная лента новостей;
- постраничная подгрузка (`pagination`);
- ручное обновление;
- экран детальной новости с открытием статьи внутри приложения.

## Настройки

Во вкладке настроек доступны:

- переключение языка: `Русский` / `English`;
- переключение темы: `Светлая` / `Тёмная` / `Как в системе`;
- изменение адреса backend-сервера;
- переход к повторному выбору института и группы.

Это позволяет тестировать приложение не только в симуляторе, но и на физическом iPhone, указывая локальный адрес backend в одной Wi‑Fi сети.

## Системные требования

- macOS с установленным `Xcode 16` или новее.
- iPhone Simulator или физическое устройство с `iOS 18.0+`.
- Подключение к интернету для загрузки новостей.
- Доступный backend для каталога институтов и расписания.

## Запуск проекта

1. Клонировать репозиторий:

```bash
git clone https://github.com/DmitryBeton/MCU-Plus-iOS.git
```

2. Открыть проект:

```bash
open MGPUPlus.xcodeproj
```

3. Выбрать схему `MGPUPlus`.
4. Запустить приложение через `Cmd + R`.

## Настройка backend для расписания

По умолчанию приложение ожидает backend, который поддерживает следующие endpoint-ы:

```http
GET /api/v1/catalog/
GET /api/v1/schedule/?groupId=1&from=2026-02-09&to=2026-02-16
```

Пример ответа каталога:

```json
{
  "institutes": [
    {
      "id": 1,
      "name": "Институт цифрового образования",
      "shortName": "ИЦО",
      "logo": "",
      "groups": [
        {
          "id": 1,
          "name": "ЦИБ-241",
          "course": 2
        }
      ]
    }
  ]
}
```

Пример ответа расписания:

```json
{
  "group": {
    "id": 1,
    "name": "ЦИБ-241",
    "course": 2,
    "instituteId": 1,
    "instituteName": "Институт цифрового образования"
  },
  "items": [
    {
      "id": 1,
      "date": "2026-02-09",
      "title": "Математический анализ",
      "teacher": "Иванов И.И.",
      "room": "101",
      "startAt": "2026-02-09T09:00:00+03:00",
      "endAt": "2026-02-09T10:20:00+03:00",
      "status": "active",
      "comment": "ЛК"
    }
  ]
}
```

Если приложение запускается на физическом iPhone, а backend поднят локально на Mac:

1. Подключить Mac и iPhone к одной Wi‑Fi сети.
2. Запустить backend на внешнем интерфейсе:

```bash
python manage.py runserver 0.0.0.0:8000
```

3. Разрешить IP-адрес Mac в Django `ALLOWED_HOSTS`, например:

```python
ALLOWED_HOSTS = ["127.0.0.1", "localhost", "192.168.1.4"]
```

4. В приложении открыть `Настройки` и указать адрес backend, например:

```text
http://192.168.1.4:8000
```

## Ограничения

- Новости загружаются через HTML-парсинг сайта, поэтому зависят от структуры страницы МГПУ.
- В проекте пока нет автоматических тестов.
- Локальные заметки и задачи не синхронизируются между устройствами.
- Устойчивость расписания зависит от доступности backend и корректности его данных.
